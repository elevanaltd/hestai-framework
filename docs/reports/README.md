# HestAI Framework Reports

**⚠️ MANDATORY TEMPLATE USAGE**

All reports in this directory MUST use the standardized template format. This ensures consistent tracking, accountability, and systematic resolution of framework issues.

## TEMPLATE REQUIREMENT

**YOU MUST USE THE TEMPLATE**: Before creating any report, copy the template structure from `00-REPORT_TEMPLATE.md` and fill in the required sections. Reports that do not follow the template format will be rejected.

## Quick Start

1. **Copy the template**: Use `00-REPORT_TEMPLATE.md` as your starting point
2. **Name your report**: Use format `REP[XXX]-[DESCRIPTIVE_NAME]-report.md`
3. **Fill all sections**: Complete metadata, findings, action items, and validation
4. **Get next report ID**: Check existing reports to determine next REP number

## Report Types for Framework

- **COMPONENT** - Component quality/structure analysis
- **OCTAVE** - OCTAVE format compliance review  
- **PATTERN** - Pattern effectiveness analysis
- **INTEGRATION** - Component integration testing
- **VALIDATION** - Framework validation review
- **ARCHITECTURE** - Framework architecture review

## Naming Convention

**Format**: `REP[XXX]-[DESCRIPTIVE_NAME]-report.md`

**Examples**:
- `REP001-octave-compliance-analysis-report.md`
- `REP002-component-integration-testing-report.md`  
- `REP003-pattern-effectiveness-review-report.md`

## Status Tracking

- **OPEN** - New report, no action taken
- **IN_PROGRESS** - Actions being implemented
- **RESOLVED** - All critical/high actions completed
- **CLOSED** - Report archived, no further action needed

## Priority Levels

- **CRITICAL** - Framework-breaking issues requiring immediate attention
- **HIGH** - Important issues affecting component quality or usability
- **MEDIUM** - Improvements that enhance framework quality
- **LOW** - Minor optimizations or nice-to-have improvements

## Assignment Roles

- **FRAMEWORK_ARCHITECT** - Framework architecture specialist
- **COMPONENT_AUTHOR** - Component creator/maintainer
- **OCTAVE_SPECIALIST** - OCTAVE format expert
- **PATTERN_DESIGNER** - Pattern design specialist
- **UNASSIGNED** - Awaiting assignment

**NO EXCEPTIONS**: Every report must follow the template format. This is not optional.

For questions about the template or reporting process, refer to the template file or contact the framework architects.
